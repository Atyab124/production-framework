## Step 3 — Conflicts identified + resolved via precedence ladder

### CF1 — Apparent conflict on F-V9 fix shape (RESOLVED at ladder step 0 — careful read)

**Surface:** WS2 Q5 said "change WHEN gate clears via UserPromptSubmit hook (only on task-shape verbs)." WS4 FM-12 said "do NOT cache verdict; gate logic untouched; display-only compression."

**Resolution:** On careful read these are NOT in conflict. Both target the `user-prompt-submit` hook. Both keep `pre-tool-use` gate logic intact. Both rely on regex-level structural detection (NOT LLM-self-attestation, which is what WS4 warns against).

WS2 prescribes: regex on human-turn content for task-shape verbs → only THEN reset `last_user_prompt_at`.
WS4 prescribes: ignore `<system-reminder>` events for `last_user_prompt_at`.

These describe the same fix from different angles. WS4 added the explicit anti-pattern guard (don't convert qualitative judgment to LLM-self-attested gates).

**Synthesized prescription:** modify `user-prompt-submit` hook to (a) detect `<system-reminder>` events and skip the timestamp write entirely, (b) detect task-shape verbs in human-turn content and write timestamp ONLY when present. Both checks are deterministic regex; neither is LLM-self-attested. Pre-tool-use gate logic untouched.

### CF2 — F-V10 Builder fix shape (RESOLVED at ladder step 2 — citation strength)

**Surface:** WS1 D1 prescribes "verb-conditional empty-diff gate" (verb in {implement, build, fix, refactor, migrate} → diff check fires). WS4 FM-13 prescribes "dispatch-time `scope: code | verdict | analysis` declaration" (gate fires only on `scope: code`).

**Both apply use-case-fit** (ladder step 1 ties).

**Citation strength check (step 2):** WS1 D1 cites 4 sources (Playwright Issue #38370 + DEV.to + Anthropic + PF F-V10 empirical) — STRONG. WS4 FM-13 cites OWASP/ACM/ITIL/SOC2 alert-fatigue research — BINDING-equivalent on the false-positive failure mode specifically.

**Resolution:** Refined synthesis. The verb-list (WS1) is the operational signal; the explicit `scope` declaration (WS4) is the contract that prevents drift. Combine both:
- Dispatch metadata MUST include `scope: code | verdict | analysis | docs`
- Gate fires only when `scope == code` AND dispatch verb in action set
- Empty diff in `scope: code` dispatch → DONE_WITH_CONCERNS auto-tag
- Empty diff in any other scope → no downgrade

This makes the gate falsifiable by both axes (verb + explicit scope) and avoids WS4's warned "convert qualitative judgment into LLM-self-attested quantitative gates" failure.

### CF3 — F-V12 fast-path threshold (RESOLVED at ladder step 2 — citation strength + adversarial framing)

**Surface:** WS2 Q7 prescribes <30 LOC AND <3 files AND no Tier 3 trigger AND in-handover-scope. WS4 FM-14 prescribes default-deny semantics + CONFIG-declared file-glob allowlist + ALL 8 Tier 3 triggers tested at gate.

- Escalation: if any Tier 3 trigger flips to true mid-execution, immediately escalate back to full plan

## Step 4 — Final synthesis paragraph

The v2.2.0 research wave converges on a clean diagnosis: PF v2 has strong PREVENTION (front-loaded gates, HARD-GATEs, citation discipline) but weak DETECTION (silent post-DONE failures) and weak ADAPTATION (no session-state model; ceremony tax doesn't scale down). The fix is two parallel layers: (1) **detection layer** = post-execution checks added to existing agents, NOT a new agent (5/5 sources prefer evaluator-optimizer pattern over adding a sixth agent), and (2) **adaptation layer** = session-state cooperation across skills (cycle-state.md marker per wrap-up feedback) + sub-agent verdict inheritance (4/4 frameworks BINDING) + fast-path with default-deny + 8-trigger test (refined from WS2/WS4 reconciliation). Recovery layer is per-tool prose first (lower cost, higher coverage), with the trigger-audit.jsonl schema extension as the discovery substrate. WS4's adversarial framing is the binding constraint: every fix must preserve the *blocking semantic* of the gate (VS-03 + VS-05 prove this is what produces correctness); fixes target the firing-rate/UX layer, never the gate-fire layer. Three new findings from wrap-up feedback fold in: F-V13 (Windows path-separator bug in docs/ auto-allow — actual shipped defect, not a feature gap), F-V9 amended (cycle-state.md marker shape proposed), VS-05 (gating ROI net-positive empirical anchor).
